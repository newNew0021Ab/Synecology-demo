Нужно переписать текущий прокси, который работает на Replit, под Netlify Functions, чтобы в продакшне (Netlify) запросы шли через serverless API и не приводили к ошибке "Server returned HTML error page instead of JSON data".  

**Требования:**

1. Создать две Netlify Functions:
   - `netlify/functions/blog.js`
   - `netlify/functions/cases.js`

2. Обе функции должны:
   - Формировать URL для Directus API, например:
     - Блог: `https://directus-production-6ce1.up.railway.app/items/blog_posts?fields=*`
     - Кейсы: `https://directus-production-6ce1.up.railway.app/items/case_studies?fields=*`
   - Сделать `fetch` к этому URL, передать параметры из `event.queryStringParameters` если есть.
   - Возвращать JSON с Directus, включая заголовки:
     ```
     {
       statusCode: 200,
       headers: {
         "Access-Control-Allow-Origin": "*",
         "Access-Control-Allow-Methods": "GET, OPTIONS",
         "Content-Type": "application/json"
       },
       body: JSON.stringify(data)
     }
     ```
   - Обрабатывать ошибку, если ответа нет или код HTTP ≠ 200, возвращать статус 500 с сообщением.

3. В `netlify.toml` прописать:
[build]
functions = "netlify/functions"

[functions]
external_node_modules = ["node-fetch"]

[[redirects]]
from = "/api/blog"
to = "/.netlify/functions/blog"
status = 200
force = true
from = "/api/cases"
to = "/.netlify/functions/cases"
status = 200
force = true
