После подключения Directus появилось много багов и нестабильностей.
Пожалуйста, выполни глубокую отладку и рефакторинг, с учётом следующих пунктов:

✅ Цель
Сайт должен:

Загружать данные из Directus без ошибок;

Отображать списки и страницы blog/case-studies корректно;

Не падать при пустых/неполных данных;

Не дублировать или некорректно отображать контент;

Не оставлять «висячих» состояний в React (ошибки из useEffect, stale state, и т.п.);

Работать стабильно при перезагрузке.

🔍 Что нужно сделать:
1. Очистить лишние зависимости
Убери неиспользуемые импорты, компоненты и хуки.

Проверь package.json на наличие лишних библиотек, особенно после подключения Directus.

2. Проверить запросы
Убедись, что fetch/SDK-запросы:

Используют правильный API URL,

Работают даже если поля null или не заданы,

Не повторяются бесконечно в useEffect.

3. Обработка данных
Убери undefined, null или ошибочные поля из рендеринга.

Проверь поля slug, cover_image, tags, seo_*, content, gallery, metrics и т.д. на наличие и корректность.

Почини отображение Rich Text (например, чтобы HTML не рендерился как текст).

4. UI и навигация
Проверь корректность маршрутов (/blog/:slug, /case-studies/:slug) — нет ли 404/пустых данных.

Отладь Link компоненты — чтобы клики из списка точно вели на нужные страницы.

Убедись, что при перезагрузке страниц slug-ов всё продолжает работать (без client-only бага).

5. Обработка ошибок
Добавь отображение ошибок при запросах (например: "Не удалось загрузить данные").

Добавь спиннеры/лоадеры для UX.

6. SEO
Убедись, что заголовки и мета-описания корректно подставляются из CMS (seo_title, seo_description).

⚠️ Особое внимание:
Проверь формат cover_image: возможно, Directus отдаёт объект, а не просто URL — нужно достать .data.id или data.full_url.

Проверь отображение массивов tags, gallery, metrics, results — не падают ли, если они пустые.

Если какой-либо slug не найден — покажи 404, а не белый экран или ошибку.

Убедись, что нет конфликтов по ключам в map().

После очистки — можешь вынести хук useDirectusCollection(name) для универсальной загрузки коллекций, если считаешь нужным.
